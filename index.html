Problem A. Symmetric Square Grid

time limit per test: 2 seconds

memory limit per test: 256 megabytes

input: standard input    output: standard output

Problem Statement

You are given n identical square tiles. Your task is to arrange them into a rectangular grid that satisfies the following conditions:

  1. The grid must have rows rows and cols columns.
  2. The number of rows must be greater than or equal to the number of columns: rows ≥ cols.
  3. The grid should be as close to a square as possible, i.e. the difference rows − cols must be minimized.
  4. If there are multiple pairs (rows, cols) with the same minimal difference, choose the one with the smallest area rows × cols.
  5. All n tiles must be placed into the grid. The remaining rows × cols − n cells must be left empty.
  6. Empty cells must be placed symmetrically according to the following rule:
    • Let empty = rows × cols − n be the number of empty cells.
    • If empty = cols − 1, all empty cells are placed in the last row (row index rows − 1).
    • Otherwise, all empty cells are placed in the second-to-last row (row index rows − 2).
    • Within that row, the empty cells must be horizontally centered: the first empty cell occupies column start_c = ⌊(cols − empty + 1) / 2⌋, and the remaining empty − 1 empty cells follow consecutively.

Output the grid, using # for filled cells and . for empty cells.

Input

The first line contains a single integer n (1 ≤ n ≤ 106) — the number of tiles to arrange.

Output

On the first line, print two integers rows and cols — the dimensions of the grid.

Then print rows lines, each containing cols characters, representing the grid according to the rules above.

Note

In the first example (n = 5), the optimal grid is 3 × 2 because 3 ≥ 2, 3 × 2 = 6 ≥ 5, and the difference 3 − 2 = 1 is minimal. Here empty = 1 = cols − 1, so the empty cell is placed in the last row, at column ⌊(2 − 1 + 1)/2⌋ = 1.

In the third example (n = 10), the grid is 4 × 3 with empty = 2 = cols − 1. The empty cells are placed in the last row, starting at column ⌊(3 − 2 + 1)/2⌋ = 1, producing #...

In the fourth example (n = 11), the grid is 4 × 3 with empty = 1 < cols − 1. The empty cell is placed in the second-to-last row (row index 2), at column ⌊(3 − 1 + 1)/2⌋ = 1, producing #.#.

A Note from the Author

If the placement rule feels a bit unusual at first glance, you're not alone. The pattern is consistent and deterministic, but putting it into words turned out to be harder than expected. Sometimes the logic is clearer when you see it in action than when you try to describe it formally.

The examples should give you a solid intuition for how the empty cells are distributed. If you've figured out the pattern from them — great. If not, don't worry: the rule is fully specified in the statement, even if it took a while to articulate.
